# /improve — レビュー→改善→PR の一気通貫

以下のワークフローを順に実行してください。

## ワークフロー

### 1. レビュー

引数が指定されている場合はその観点でレビューする。
引数がない場合は以下の3つのレビューを並列サブエージェントで実行する
（各サブエージェントに対応する `.claude/skills/review-*.md` の手順を渡す）:

- `review-swift-code` スキル — コーディング規約レビュー
- `review-swift-tests` スキル — テストコードレビュー
- `review-architecture` スキル — アーキテクチャレビュー

### 2. 改善提案

レビュー結果を統合し、改善項目を優先度順にリストアップする。
ユーザーに改善内容を提示し、実装して良いか確認する。

### 3. 実装

承認された改善を実装する。
独立したタスクはサブエージェントで並列実行する。
各変更後に `swift build` と `swift test` で検証する。

### 4. コミット・PR

すべての改善が完了したら:
1. 変更をコミットする（Conventional Commits + 日本語）
2. ユーザーに PR 作成の要否を確認する
3. 承認されたら PR を作成する

引数: $ARGUMENTS
