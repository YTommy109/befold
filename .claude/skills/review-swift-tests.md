## Review Swift Tests

`docs/coding_rule.md` のテスト規約に基づいて Swift テストコードをレビューする。

### 使い方

引数としてファイルパスまたはディレクトリパスを受け取る。
引数がない場合は `MmdviewApp/mmdviewTests/` 配下の全 `.swift` ファイルを対象にする。

### Steps

1. `docs/coding_rule.md` を読み込み、テスト規約を把握する。
2. 対象ファイルを特定する:
   - 引数にファイルパスがあればそれを使う
   - 引数にディレクトリがあれば配下の `.swift` ファイルを列挙する
   - 引数がなければ `MmdviewApp/mmdviewTests/` 配下を列挙する
3. 対応するプロダクトコードも読み込み、テストの網羅性を判断する。
4. 各ファイルを以下のチェックリストに沿ってレビューする。
5. 発見事項をファイルごとに報告する。

### チェックリスト

#### テストフレームワーク

- [ ] `import Testing` を使っているか（`import XCTest` は不可）
- [ ] アサーションに `#expect` を使っているか（`XCTAssert*` は不可）

#### テスト構造

- [ ] テストスイートが `@Suite` + `struct` で定義されているか
- [ ] 各テスト関数に `@Test` が付いているか
- [ ] `@MainActor` 隔離が必要なテストにスイートレベルで `@MainActor` が付いているか

#### テスト関数の命名

- [ ] 関数名がテスト対象の振る舞いを表す英語の lowerCamelCase か
- [ ] 曖昧な名前（`test1` など）や過度に長い名前がないか

#### パラメタライズ

- [ ] 同じアサーション構造で入力だけが異なるテストが `@Test(arguments:)` でまとめられているか
- [ ] 手動で `#expect` を並べているだけのテストがないか（最初の失敗で残りが検証されない）
- [ ] パラメタライズすべき典型的なケースを見逃していないか:
  - 複数の入力に対して同じ変換結果を期待する
  - 複数の無効入力に対して同じエラーを期待する
  - 境界値テスト

#### ファイルシステムテスト

- [ ] 一時ディレクトリを `UUID` 付きで作成しているか（テスト間の独立性）
- [ ] `defer` でクリーンアップしているか
- [ ] `FileManager.default.temporaryDirectory` を使っているか（固定パス不可）

#### 非同期テスト

- [ ] コールバックテストに `confirmation { confirm in ... }` を使っているか
- [ ] `@Test(.timeLimit(.minutes(1)))` でタイムアウトが設定されているか
- [ ] `nonisolated(unsafe)` はテストコード内のミュータブル変数のみに使われているか

#### コメント

- [ ] テスト内のブロックが日本語コメントで区切られているか（前提条件・操作・検証の意図）
- [ ] 関数名だけでは意図が伝わらないテストに `///` で補足があるか
- [ ] 不要なコメント（コードをそのまま訳しただけ）がないか

#### テスト網羅性

対応するプロダクトコードを読み、以下を確認する:

- [ ] 公開メソッドにテストがあるか
- [ ] 正常系・異常系の両方をカバーしているか
- [ ] 状態遷移（例: `isDeleted` の変化）のテストがあるか
- [ ] エッジケース（空文字列、存在しないファイル、大文字/小文字）がカバーされているか

#### コード品質

- [ ] テストコードの行長が 120 文字以下か
- [ ] テスト関数が肥大化していないか（目安: 30 行以内）
- [ ] テスト間で共通のセットアップが `private func` に抽出されているか

### 出力フォーマット

ファイルごとに発見事項を以下の形式で報告する:

```
## <ファイルパス>

### 問題 (問題がある場合)

- **<カテゴリ>**: <具体的な指摘> (L<行番号>)

### 改善提案 (改善できる場合)

- <パラメタライズ化の提案、不足しているテストケースなど>

### 良い点 (特筆すべきものがある場合)

- <規約に沿った良い実装の具体例>
```

最後に全体のサマリーを付ける:

```
## サマリー

- レビュー対象: <N> ファイル (<N> テスト関数)
- 問題: <N> 件（重大: <N>, 軽微: <N>）
- テスト網羅性: <高/中/低>
- 不足しているテストケース: <具体的なリスト>
- 規約準拠度: <高/中/低>
```
