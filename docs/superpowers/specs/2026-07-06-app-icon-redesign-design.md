# befold アプリアイコン刷新 — 設計

## 背景・目的

現行アイコン（[BefoldApp/AppIcon/icon.svg](../../../BefoldApp/AppIcon/icon.svg)）は、
矢印でつながった 3 ノードのフローチャート＝ Mermaid 図のメタファーで作られている。
しかしアプリは `.mmd` / `.md` だけでなく **図表ファイルとソースコードファイルを含む
大量のファイル** を閲覧する汎用ビューアへと役割が広がり、フローチャート表現が
実態と合わなくなった。

新アイコンは次を象徴する:

- **behold（覗く／眺める）** = 虫眼鏡（ガラスレンズ）を主役にする
- **大量のファイル** = 背後に重なる複数の書類カード
- **図表もコードも見る** = カード内に図表ノードとシンタックスカラーのコード行を同居

macOS らしい上質さのため、暗色系の背景に **SVG でエミュレートした Liquid Glass 風**
のレンズを載せる（真の Icon Composer レイヤー形式は今回は採用しない）。

## 採用案: 案B「ヒーローレンズ」

ブレインストーミングで 3 案（A: ファン状スタック / B: ヒーローレンズ / C: 分割ドキュメント）を
提示し、案B を採用。ガラスのレンズを大きく主役に据え、その下で図表＋コードを拡大表示する。
Liquid Glass 感が最も出て、16px でも「虫眼鏡」と即読めるシルエットを持つ。

## 構図（viewBox 0 0 512 512, 角丸 rx 112）

下から上のレイヤー順:

1. **背景スクワークル** — 暗色系の対角グラデーション（上明→下暗）＋上部の柔らかな
   ラジアルグロー。Mac アプリらしい奥行き。
2. **書類スタック（背後）** — わずかに回転・オフセットした 2 枚のカードを前面カードの
   背後にのぞかせ、「大量のファイル」を示唆。
3. **前面カード** — 上半分に図表（3 ノード＋コネクタ）、下半分にシンタックスカラーの
   コード行（3 本）。図表ファイルとコードファイルの両方を 1 枚に凝縮。
4. **虫眼鏡（主役）**
   - **ハンドル**: 45° に伸びる角丸バー。金属質のリニアグラデーション。
   - **拡大コンテンツ**: レンズ円で `clipPath` した、下のコード行を拡大・明色化した像。
     「ガラス越しに拡大して見ている」質感の要。
   - **ガラス本体**: 白の弱いラジアルグラデーション（中心上寄りが明るい半透明）。
   - **リム**: 左上が明るい金属質グラデーションの太いストローク＋内側に細い暗色リング。
   - **スペキュラ**: 左上に白い弧のハイライト（ガラスの反射）。

## パイプライン（現行維持）

`AppIcon/icon.svg` を差し替え、[BefoldApp/scripts/build_icon.sh](../../../BefoldApp/scripts/build_icon.sh)
で `befold/Resources/AppIcon.icns` を再生成する。ビルド構成・`project.yml` の変更は不要。

<!-- constrained-by ../../../BefoldApp/scripts/build_icon.sh -->

**注意（要検証）**: この環境に `rsvg-convert` は無く、ビルドは `qlmanage`（WebKit）
フォールバックでラスタライズする。`clipPath` と複数グラデーションが qlmanage で正しく
描画されるかは環境依存のため、実装時に生成 PNG を実機で目視確認する。崩れる場合の
代替: (a) `clipPath` を使わずレンズ内像を単純な明色バーで近似する、(b) Homebrew の
`librsvg` を導入して `rsvg-convert` 経路に切り替える。

## 受け入れ基準

- `build_icon.sh` が成功し `AppIcon.icns` が更新される。
- 16 / 32 / 128 / 256 / 512 px（および @2x）で破綻なくラスタライズされる。
- **16px**: 虫眼鏡のシルエットと背後のカードが判別でき、単なる塊にならない。
- **128px 以上**: レンズ内の拡大コード行・スペキュラ・書類の図表／コードが読み取れる。
- Finder / Dock で表示し、暗色背景と明色書類のコントラストが十分。

## 非対象（YAGNI）

- Icon Composer による真の Liquid Glass レイヤー化（将来の別タスク）。
- アプリ内 UI・スプラッシュ等アイコン以外の意匠。
- ライト/ダーク別アイコンの出し分け。
