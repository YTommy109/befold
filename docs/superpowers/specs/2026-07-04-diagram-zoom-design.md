# ダイアグラム個別ズーム 設計書

<!-- derived-from ./2026-06-06-zoom-design.md -->

Date: 2026-07-04

## 概要

現在のズームボタン（右上フローティングパネル）は、メニュー / Cmd± と同じく
`#diagram-wrap`（コンテンツ全体）を拡縮している。これを次の役割分担に変更する。

| 操作 | 対象 |
|------|------|
| メニュー（View > Zoom In/Out/Reset）/ Cmd± | HTML 全体ズーム（現状維持） |
| ダイアグラムのホバーボタン | そのダイアグラム個別のズーム（新規） |
| Ctrl+ホイール / トラックパッドピンチ | ポインタ直下がダイアグラムならそのダイアグラム、それ以外は全体ズーム |

markdown 表示ではページ全体用のズームボタンは表示しない。markdown 内の
`mermaid` フェンスブロックにも、.mmd ファイルのダイアグラムにも、同じホバー
ボタンが付く。

## UI 設計

### ホバーコントロール

<!-- supersedes ./2026-06-06-zoom-design.md#ズームコントロールパネル -->

右上の `position: fixed` フローティングパネルは**削除**し、代わりに各ダイアグラム
（`.mermaid` 要素）のラッパー右上に同じ見た目の `[ − 100% + ]` コントロールを
`position: absolute` で重ねる。

- 通常時は非表示（`opacity: 0`）。ラッパーへのマウスホバーでフェード表示
- `−` / `+` ボタン: 25% 刻みで拡縮
- `%` ラベル: クリックで 100% にリセット（既存と同じ操作系）
- ガラス調スタイル（背景ブラー・角丸・影）は既存 `.zoom-controls` の定義を流用

### はみ出し処理（枠固定・中スクロール）

ダイアグラムの外枠は 100% 時のサイズで固定し、拡大時は枠内をスクロール（パン）
できるようにする。markdown 本文のレイアウトはダイアグラムのズームで動かない。

```
拡大前:                拡大後 (150%):
┌─ mermaid ───┐      ┌─ mermaid ───┐
│ ◇→□→○      │      │ ◇──→□── →  │ ← 枠サイズ不変
│            │      │            │    中をスクロール
└────────────┘      └────────────┘
本文は動かない        本文は動かない
```

.mmd ファイルではラッパーがビューポートいっぱいに広がるため、実質「ウィンドウ内
スクロール」になる。

## ズーム仕様

| 項目 | 全体ズーム | ダイアグラムズーム |
|------|-----------|------------------|
| 範囲 | 50%〜200%（現状維持） | 50%〜300% |
| ステップ | 25% 刻み | 25% 刻み |
| リセット | メニュー / % ラベル | % ラベルクリック |
| 永続化 | ファイル毎に UserDefaults（現状維持） | なし（セッション内のみ） |
| 再レンダリング | 維持（現状維持） | 維持（ブロック順インデックスをキーに JS 内で保持） |

ダイアグラムズームはウィンドウを閉じると失われる。markdown 編集でダイアグラムの
順番・個数が変わった場合、インデックスがずれてズーム値が別のダイアグラムに付く
ことは許容する（永続化しないため影響はセッション内に限定される）。

## 実装方式

### ラッパー構造（レンダリング後に JS で生成）

`render()` の mermaid 描画後、各 `.mermaid` 要素を以下で包む。

```html
<div class="diagram-zoom-wrap">          <!-- position:relative; overflow:auto;
                                              高さ=100%時の実測値で固定 -->
  <div class="diagram-zoom-inner">        <!-- CSS zoom の適用対象 -->
    <pre class="mermaid">…(SVG)…</pre>
  </div>
  <div class="diagram-zoom-controls">     <!-- ホバーで表示 -->
    <button>−</button><span>100%</span><button>+</button>
  </div>
</div>
```

- CSS `zoom` を使う理由は既存設計と同じ: `transform: scale()` と違いレイアウト
  サイズが変わるため、ラッパーの `overflow: auto` でスクロール領域が自然に追従する
- ラッパーの高さは mermaid 描画完了後に 100% 時の実測値で固定する。
  .mmd（ダイアグラム 1 個・全画面）ではビューポート高を上限とする
- ズーム値は `Map<index, zoom>` で JS に保持し、再レンダリング時にインデックスで
  再適用する

### Ctrl+ホイール / ピンチの分岐

macOS トラックパッドのピンチは `ctrlKey: true` の `wheel` イベントとして届く
（既存実装と同じ仕組み）。ハンドラで `event.target.closest('.diagram-zoom-wrap')`
を判定し、ダイアグラム上ならそのダイアグラムのズーム、それ以外なら既存の全体
ズーム（`_mmdWheelZoom`）に振り分ける。

### 全体ズーム（変更最小）

- `_mmdZoomIn` / `_mmdZoomOut` / `_mmdZoomReset` / `_mmdApplyZoom` と
  `#diagram-wrap` への CSS `zoom`、ZoomStore への永続化ブリッジは**そのまま**
- 変更点は「右上固定パネルの HTML / CSS を削除する」ことのみ
- 上限・下限・ステップの共通定数（`ZOOM_MIN` / `ZOOM_STEP` 等）と
  `clampZoom` / `stepZoom` はダイアグラムズームでも共用する
  （ダイアグラム側の上限のみ 3.0 の別定数）

## 変更ファイル

| ファイル | 変更内容 |
|----------|----------|
| `MmdviewApp/mmdview/Resources/viewer.html` | 固定パネル削除・ラッパー生成とダイアグラムズームの JS・ホイール分岐 |
| `MmdviewApp/mmdview/Resources/viewer.js` | ダイアグラムズーム用の純粋関数（clamp/step の共用化・上限定数） |
| `MmdviewApp/mmdview/Resources/style.css` | `.zoom-controls` の fixed 配置を削除し、ホバーコントロール用スタイルに置換 |

Swift 側（ZoomStore / MainMenuBuilder / ViewerWindowController / ViewerWebView）
は変更しない。

## テスト

- `viewer.js` の純粋関数（clamp / step）は既存のテスト方式があればそれに追従
- WebView 上の挙動（ホバー表示・枠内スクロール・ホイール分岐・再レンダリング後の
  ズーム維持）は `/webview-smoke` および手動確認:
  1. .mmd ファイル: ホバーボタンで拡大 → 枠内スクロールできる
  2. mermaid ブロックを 2 個含む .md: 個別にズームでき、本文レイアウトが動かない
  3. Cmd± で全体ズーム（markdown 本文も拡縮）、再起動で復元される
  4. ダイアグラム上とその外での Ctrl+ホイールの振り分け
  5. ファイル保存 → 自動再描画でダイアグラムズームが維持される

## 非変更範囲

- Swift コード全般（メニュー・ショートカット・永続化）
- markdown レンダリング（markdown-it のフェンスルール含む）
- mermaid レンダリングパイプライン（`mermaid.run()` の呼び方）
