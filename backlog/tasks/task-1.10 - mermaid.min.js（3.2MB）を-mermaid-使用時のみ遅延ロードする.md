## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
パフォーマンスレビュー（2026-07-18）で発見（推奨）。viewer.html:53 で <script src=mermaid.min.js> を無条件・同期ロードしており、CSV・ログ・コード・SVG・HTMLソース等 mermaid 不使用のプレビューでも 3.2MB の JS を毎回パース・評価する（markdown-it 121KB / highlight 124KB と桁違い）。WKWebView 初期化+viewer.html ロードの起動コスト支配項で、QuickLook の『数百ms で出す』目標に重い。.mmd もしくは mermaid ブロックを含む markdown を描画する時だけ動的ロードする。
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [x] #1 mermaid を使わない種別のプレビューで mermaid.min.js がロード/評価されない
- [x] #2 .mmd および mermaid ブロック入り markdown で従来どおり図が描画される
- [x] #3 初期表示までの時間が mermaid 不使用ケースで改善する
<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. viewer.html の <script src=mermaid.min.js> 同期ロードを削除する
2. _mmdEnsureMermaidLoaded() を追加し、初回必要時にのみ <script> をDOM挿入してロードするPromiseキャッシュとする
3. mermaid.initialize() / mermaid.parseError の設定をロード完了後のコールバックに移す
4. render() の mermaidElements 検出ブロック（type==='mmd' 用の <pre class=mermaid> も markdown 内 mermaid フェンスも共通でここを通る）で、要素が1つ以上あるときだけ _mmdEnsureMermaidLoaded() を await してから mermaid.run() する
5. ダークモード変更リスナーは mermaid が未ロードなら re-initialize をスキップする（次回ロード時に最新configで初期化されるため）
6. swift build / swift test で回帰がないことを確認し、実機（/run）で .mmd・mermaidブロック入りmd・mermaid不使用mdの3パターンを目視確認する
<!-- SECTION:PLAN:END -->

## Implementation Notes

<!-- SECTION:NOTES:BEGIN -->
実装: viewer.html から <script src=mermaid.min.js> の同期ロードを削除し、_mmdEnsureMermaidLoaded()（Promiseキャッシュ付き動的<script>挿入）を追加。mermaid.initialize()/mermaid.parseError の設定をロード完了コールバックへ移動。render() 内の mermaidElements 検出ブロック（.mmd の <pre class=mermaid> と markdown 内 mermaid フェンスの両方がここを通る）でのみ、要素が1つ以上あるときに await _mmdEnsureMermaidLoaded() する。ダークモード変更リスナーは _mermaidLoadPromise が存在する場合のみ re-initialize する。
検証: swift build 成功、swift test 384件全て成功（回帰なし）。/run でビルドした実機で .mmd / mermaidブロック入りmd / mermaid不使用md の3パターンを目視確認し、いずれも正しく描画されることをスクリーンショットで確認。AC1（非ロード）は、mermaid.min.js への参照が render() 内の mermaidElements.length>0 ガード配下の _mmdEnsureMermaidLoaded() 呼び出し1箇所のみであり、csv/code/image/pdf/html/plain markdown の各分岐はこの関数を呼ばない、というコード構造上の検証で裏付けた（fs_usage 等の実測は sudo 権限の都合で見送り、ユーザー確認の上でコード検査+目視確認を採用）。
<!-- SECTION:NOTES:END -->
